Bem-vindo a API da MShop!

Essa é a página raiz da API. Se quiser acessar, modificar, ou deletar dados, utilize os seguintes caminhos:

/clients

  • Retorna todos os clientes.
  • Retorna um específico cliente se usado desta forma: /client/:id. Ex.: /client/1 retorna o cliente de ID 1.
  • Aceita POST e DELETE requests, onde o ID do usuário precisa ser especificado no DELETE.
  • No POST request, é necessário informar as características "name" e "cpf".

/products

  • Retorna todos os produtos.
  • Pode retornar um específico produto se usado o ID do produto como parâmetro. Ex.: /products/1 retorna o produto de ID 1.
  • Como em /clients, aceita DELETE e POST requests da mesma forma.
  • No POST request, é necessário informar as seguintes características: "name", "description", "price" e "type."

/cart/:id

  • Retorna o carrinho de determinado usuário.
  • Utilize o ID do usuário no lugar de ":id".
  • Para adicionar um novo item ao carrinho com um POST request, informe no body as seguintes características: "productID", como o ID do produto a ser adicionado, e "date" sendo a data da compra, no formato yyyy-mm-dd.
  • Para deletar um item do carrinho, utilize o ID da compra em si como listada no banco de dados quando fizer um DELETE request.

/address

  • Retorna os endereços cadastrados pelo usuário.
  • Você pode realizar GET, POST e DELETE requests utilizando IDs individuais como nos outros caminhos. Um GET request para /address/1 retorna o endereço do usuário 1, por exemplo.
  • Para adicionar um endereço de usuário novo com um POST request, utilize as seguintes características no body:
    1. "userId" sendo o ID do usuário.
    2. "road" sendo a rua a cadastrar.
    3. "number" sendo o número do endereço.
    4. "city" sendo a cidade do usuário.
    5. "county" sendo o município do usuário.
    6. "neighborhood" sendo o bairro do usuário.
    7. "complement" sendo o complemento do endereço. Opcional.
  • Para apagar com um DELETE request, utilize o ID do endereço no banco de dados, e não o ID do usuário no seguinte modo: /address/:id.
